这个页面给站长自己看，用来把站点后台配完整。

你已经有了什么

  • /admin/ 在线编辑入口
  • GitHub Pages 自动构建部署工作流
  • 文章内容模型已经绑定到 source/_posts
  • 图片上传目录已经固定为 /img/uploads

还需要你手动完成的 4 步

1. 创建 GitHub OAuth App

在 GitHub Developer Settings 里新建一个 OAuth App：

  • Homepage URL：https://sheepo2.github.io
  • Authorization callback URL：https://你的认证服务域名/api/callback

创建后记下：

  • Client ID
  • Client Secret

2. 部署一个免费的认证中转服务

Decap CMS 在 GitHub Pages 上登录 GitHub，通常需要一个轻量认证中转。

推荐二选一：

  • Cloudflare Workers
  • Vercel Functions

这个服务只做 GitHub OAuth 换 token，不存文章内容。

到 GitHub 仓库设置中把 Pages Source 改成：

GitHub Actions

之后无论是你在 /admin/ 改文章，还是直接推代码，都会自动构建发布。

权限说明

  • 访客访问公开站点时只能浏览和留言
  • 访客即使打开 /admin/，没有仓库写权限也不能增删文章
  • 真正能改文章的人，仍然是拥有该 GitHub 仓库写权限的账号
